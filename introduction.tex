\section{Introduction}
Developing a  real-life application is a part of the requirements for CSC-223.  This report describes a control file that will be used to create, populate, and query a simple  persistent client relationship management (\textbf{CRM}) system.  How the control file is created is outside the scope of this report.

The control file is a simple ASCII text file populated with assorted creation, retrieval, update, and deletion (\textbf{CRUD}) database directives.  CRUD related tokens are in plain text, while data associated with the tokens may be in plain text, or base64\footnote{Base64 is a binary-to-text encoding that uses 64 printable characters to represent each 6-bit segment of a sequence of byte values\cite{wikipedia:base64}.} encoded strings.
