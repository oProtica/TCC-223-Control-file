\section{Conclusion}
The CSC-223 real-life application is to create, populate, and utilize a  persistent client relationship management (\textbf{CRM}) system.  The design of the CRM is to have a control file manage the creation, retrieval, update, and deletion (CRUD) database operations.  Using a control file simplifies  testing the CRM by ensuring consistency and reproduciability.  This report details the specifications of the CRM control file, and how the different CRUD directives will be interpreted.

Sample Java source files that maybe of use in the creation of the CRM are included, as well as a sample control file are available in a GitHub repository.
